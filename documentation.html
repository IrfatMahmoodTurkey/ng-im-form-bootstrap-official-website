1. Introduction

NG IM FORM is a powerful, front-end library designed for developers who need to implement dynamic form creation capabilities within their applications. It provides a visual interface that allows non-technical users to build complex forms at runtime, fully powered by the responsive design of **Bootstrap 5**.

Key Benefit: Total control over form structure and immediate access to clean, structured JSON data upon submission, streamlining integration with any backend system.

2. Installation & Setup

Getting started with NG IM FORM involves installing the package and setting up the basic prerequisites.

2.1 Installation via npm or yarn

Use your preferred package manager to add the library to your project dependencies: 


# Using npm
npm install ng-im-form-bootstrap

# Using yarn
yarn add ng-im-form-bootstrap

2.2 Prerequisites

Ensure you have the following in your project setup, as the component relies on the Bootstrap framework for its styling and core functionality:

  • Bootstrap 5: CSS and JavaScript bundles must be included.
  • Angular/React/Vue: The component is framework-agnostic but is typically integrated into a modern JavaScript application.

2.3 Initialization Example (Conceptual)

After installation, import the module and initialize the form builder on your target element (e.g., within an Angular component or React hook): 


// TypeScript/JavaScript module import
import { FormBuilder } from 'ng-im-form-bootstrap';

// Assume 'containerElement' is the DOM element where the builder will live
const containerElement = document.getElementById('form-builder-root');

if (containerElement) {
    const builder = new FormBuilder({
        target: containerElement,
        initialConfig: { sections: [], fields: [] }
    });

    // Render the visual interface
    builder.render();
}

3. Key Concepts

The form structure revolves around two primary components: Sections and Form Fields.

3.1 Multiple Sections

Forms can be organized into one or more named sections. This modular approach helps manage complexity for large surveys or data entry systems, logically grouping related fields.

Visual Component:

Sections are typically rendered as collapsible panels or distinct tabs in the user interface.

3.2 Form Fields

Within each section, users can insert various field types to capture different kinds of data:

  • Text Inputs: Standard single-line data (e.g., Name, Email).
  • Text Areas: Multi-line text for detailed responses or comments.
  • Select Boxes: Drop-down menus for selecting one option from a predefined list.
  • Radio Button Groups: For selecting a single option from a small group of choices.

4. Usage

4.1 Drag & Drop Functionality

The library provides total flexibility through an intuitive drag-and-drop mechanism. Users can rearrange both the major structural elements and the individual fields:

  • Reorder Sections: Change the sequence of major form groups.
  • Reorder Fields: Adjust the placement of fields within their respective sections.

4.2 JSON Submission

The form's configuration and the submitted data are consistently formatted into JSON, ensuring reliable data handling.

Example JSON Output:


{
  "form_id": "survey-2025-q1",
  "sections": [
    {
      "section_id": "personal_info",
      "fields": [
        {
          "field_name": "fullName",
          "field_type": "text",
          "value": "Jane Doe"
        },
        // ... other fields
      ]
    },
    // ... other sections
  ],
  "submission_timestamp": "2025-10-27T10:00:00Z"
}
